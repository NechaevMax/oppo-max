АРХИТЕКТУРНЫЕ РЕШЕНИЯ
═══════════════════════════════════════════════════════════════════════════

1. ИЕРАРХИЯ ИСКЛЮЧЕНИЙ
   Вместо простого error handling используется иерархия исключений:
   
   MenuException (абстрактный базовый класс)
   ├─ FileException      - для ошибок работы с файлами
   ├─ ParseException     - для ошибок парсинга строк
   └─ ValidationException - для ошибок валидации данных
   
   ПРЕИМУЩЕСТВА:
   - Специфичная обработка разных типов ошибок
   - Возможность глобального catch(MenuException)
   - Информативные сообщения об ошибках
   - Удобство в расширении (можно добавить новые типы ошибок)

2. ОТДЕЛЕНИЕ КЛАССА MenuFileReader
   Вместо размещения логики парсинга в MenuManager:
   
   ДО: MenuManager::readFromFile() → 50+ строк кода
   ПОСЛЕ: MenuFileReader::parseMenuItem() → специализированный класс
   
   ПРЕИМУЩЕСТВА:
   - Разделение ответственности (Single Responsibility)
   - Возможность тестировать парсинг отдельно
   - Возможность переиспользования в других местах
   - Более понятный и читаемый код

3. ВАЛИДАЦИЯ НА УРОВНЕ КЛАССА
   Вместо проверок в MenuManager:
   
   ДО: Проверка в addItemManually()
   ПОСЛЕ: Проверка в конструкторе MenuItem и в setters
   
   ПРЕИМУЩЕСТВА:
   - Невозможно создать некорректный объект
   - Валидация применяется везде, где используется класс
   - Более безопасный код

КЛАССЫ И МЕТОДЫ
═══════════════════════════════════════════════════════════════════════════

КЛАСС: MenuItem
───────────────────────────────────────────────────────────────────────────
Назначение: Представление одного элемента меню (блюда)
Члены: name_ (string), price_ (double), cooking_time_ (int)
Методы:
  - MenuItem()                      - конструктор по умолчанию
  - MenuItem(string, double, int)   - конструктор с параметрами
  - validate()                      - проверка корректности
  - getName(), getPrice(), getCookingTime()  - getters
  - setName(), setPrice(), setCookingTime()  - setters с валидацией
  - print()                         - вывод информации


КЛАСС: MenuFileReader
───────────────────────────────────────────────────────────────────────────
Назначение: Парсинг строк файла меню
Статический класс (не требует создания объекта)
Методы:
  - parseMenuItem(string) static    - парсит одну строку
  - removeQuotes(string) private    - удаляет кавычки


КЛАСС: MenuManager
───────────────────────────────────────────────────────────────────────────
Назначение: Управление коллекцией блюд
Члены: items_ (vector<MenuItem>)
Методы:
  - readFromFile(string)            - загрузка из файла
  - printAll() const                - вывод всех блюд
  - sortByCookingTime()             - сортировка
  - filterByTime(int) const         - фильтрация
  - addItemManually()               - добавление вручную
  - displayMenu()                   - главное меню


ИЕРАРХИЯ ИСКЛЮЧЕНИЙ:
───────────────────────────────────────────────────────────────────────────
MenuException (базовый класс)
├─ FileException               - ошибка при открытии файла
├─ ParseException              - ошибка парсинга строки
└─ ValidationException         - ошибка валидации данных
